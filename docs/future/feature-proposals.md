## 一、小型功能（1-3 天）

### 1.1 ~~图表版本历史~~ ✅

**描述**：为每个对话记录图表代码的每次变更版本，支持回退到任意历史版本。

**实现思路**：
- 在 `messages` 表中，assistant 消息已保存了每次生成的代码
- 在 UI 层添加版本切换功能：点击历史消息中的"查看此版本"按钮，将画布切换到该版本的代码
- 右侧抽屉面板（360px），显示所有版本
- 每个版本使用对应格式的 SVG 预览（Excalidraw/Mermaid/Draw.io 各自的导出 API）
- IntersectionObserver 懒加载预览
- DOMPurify 防 XSS
- 快捷键 Alt+V 打开/关闭

**价值**：用户经常需要对比不同版本的图表，或回退到之前的效果。

**工作量**：1-2 天 ✅ 已完成

---

### 1.2 消息置顶/书签

**描述**：允许用户对重要的消息添加书签或置顶。

**实现思路**：
- 在 `messages` 表添加 `is_bookmarked INTEGER DEFAULT 0` 字段
- 在 `MessageBubble` 的操作按钮组中添加书签图标
- 在 `AICopilotPanel` 顶部添加"仅显示书签"的筛选开关

**价值**：长对话中用户需要快速定位关键的生成结果。

**工作量**：1 天

---

### 1.3 ~~模板收藏夹~~ ❌ 移除

> 理由：图表由 LLM 动态生成，模板本质是 prompt + 示例代码，与 LLM 直接对话生成无显著差异，收藏模板的价值有限。

---

### 1.4 图表代码片段库

**描述**：用户可以保存常用的图表代码片段（如自定义的样式配置、常用节点模板），在生成时作为参考注入上下文。

**实现思路**：
- 新建 `snippets` 表：`id, name, format, code, created_at`
- 在 `AICopilotPanel` 输入框旁添加"插入片段"按钮
- 片段内容作为 system message 的一部分发送给 LLM

**价值**：用户可能有固定的样式偏好或常用组件，每次都要重新描述。

**工作量**：2 天

## 二、中型模块（1-2 周）

### 2.1 ~~图表模板市场~~ ❌ 移除

> 理由：与模板收藏夹同理，AI 图表工具的核心价值在于自然语言描述 → LLM 生成，预置模板库与这一核心价值定位冲突。模板的功能可以通过更好的 prompt 引导和代码片段库替代。

---

### 2.2 代码编辑器双向同步

**描述**：代码编辑器中的修改能实时反映到画布，画布上的操作也能更新代码。

**实现思路**：
- **代码→画布**：Monaco Editor 的 `onChange` 事件，debounce 500ms 后调用 `tryParseAndApply`
- **画布→代码**：Excalidraw 的 `onChange` 回调，将 elements 序列化回 JSON 更新 `generatedCode`
- 需要处理循环更新问题：设置 `updateSource` 标记区分来源

**价值**：当前代码编辑器是单向的（代码→画布），用户在画布上的修改无法同步回代码，导致重新生成时丢失手动调整。

**工作量**：1-2 周

---

### 2.3 图表对比视图

**描述**：支持左右对比两个版本的图表，或对比不同格式的渲染效果。

**实现思路**：
- 在编辑器中添加"对比模式"按钮
- 左侧显示版本 A，右侧显示版本 B
- 版本可以是同一对话的不同消息，也可以是不同格式的渲染结果
- 使用 `SplitPane` 组件实现左右分栏

**价值**：用户在调整图表时需要对比前后效果，当前只能靠记忆。

**工作量**：1 周

---

### 2.4 批量生成

**描述**：支持一次输入多个描述，批量生成多张图表。

**实现思路**：
- 在 `AIPromptBox` 中添加"批量模式"开关
- 批量模式下，输入框支持多行（每行一个描述）
- 使用 `Promise.all` 并行调用 `/api/generate`
- 结果以网格卡片形式展示，用户可逐个查看和编辑

**价值**：当用户需要为文档生成多张图表时，逐个生成效率低。

**工作量**：1 周

---

### 2.5 自定义 AI 操作

**描述**：允许用户创建自定义的 AI 操作（类似自定义 prompt）。

**实现思路**：
- 新建 `custom_actions` 表：`id, name, icon, system_prompt, user_prompt_template, format`
- 在 `FloatingAIActions` 中添加"+"按钮，打开自定义操作编辑器
- 用户可以定义操作名称、系统提示词、用户提示词模板
- 模板支持变量插值：`{{currentCode}}`、`{{chartType}}`、`{{format}}`

**示例自定义操作**：
- "添加图例" — 在图表右下角添加颜色图例
- "转换为英文" — 将图表中的中文文字翻译为英文
- "添加标题" — 在图表顶部添加大标题
- "调整配色" — 将配色方案改为指定的色板

**价值**：当前 4 个 AI 操作是固定的，用户无法根据自己的需求定制。

**工作量**：1-2 周

---

### 2.6 多模型协作

**描述**：支持在同一对话中切换不同的 LLM 配置，或使用多模型协作生成。

**实现思路**：
- 在 `AICopilotPanel` 的输入框旁添加模型选择器（快速切换，不改变全局配置）
- 支持"模型对比"模式：同一 prompt 发送给两个模型，左右对比结果
- 高级模式：使用模型 A 生成初稿，模型 B 进行优化

**价值**：不同模型在不同图表类型上表现不同，用户可能想用 Claude 生成架构图、用 GPT-4 生成流程图。

**工作量**：1-2 周

---

### 2.7 图表注释与标注

**描述**：支持在图表上添加文字注释、箭头标注、高亮区域。

**实现思路**：
- 对于 Excalidraw：直接使用其原生的注释功能（text、arrow、highlight）
- 对于 Mermaid/Draw.io：在画布上叠加一个注释层（独立的 canvas/div）
- 注释数据存储在 `conversations` 表的 `annotations` 字段

**价值**：用户在评审图表时需要标注问题区域或添加说明。

**工作量**：1 周

---

### 2.8 导出为代码片段

**描述**：将图表导出为可在其他项目中直接使用的代码片段。

**支持的导出格式**：
| 格式 | 说明 |
|------|------|
| React 组件 | 导出为 `<Excalidraw>` 或 `<Mermaid>` React 组件代码 |
| HTML 页面 | 导出为独立的 HTML 文件，包含渲染脚本 |
| Markdown | 导出为带 Mermaid 代码块的 Markdown 文件 |
| Vue 组件 | 导出为 Vue SFC |

**价值**：开发者生成图表后需要嵌入到自己的项目中，当前只能手动复制代码。

**工作量**：1 周

---

### 2.9 快速编辑（选中元素修改）

**描述**：支持选中图表中的某个元素，通过自然语言指令修改其属性。

**实现思路**：
- Excalidraw 有 `getSelectedElements` API
- 选中元素后，输入框变为"修改选中元素"模式
- 用户输入指令如"改成红色"、"加大字号"、"移到左边"
- 发送给 LLM 时附带选中元素的 JSON 和用户指令

**示例**：
```
用户选中了一个矩形节点
输入："改成圆角蓝色"
→ LLM 返回修改后的元素 JSON
→ 应用到画布
```

**价值**：当前修改图表必须通过自然语言描述全局修改，无法精确操作单个元素。

**工作量**：1-2 周

## 三、大型模块（2-4 周+）

### 3.1 Agent 多步生成

**描述**：引入 Agent 工作流，支持多步推理和工具调用来生成复杂图表。

**核心思路**：

传统模式：用户描述 → LLM 一次性生成完整代码 → 渲染

Agent 模式：用户描述 → Agent 规划 → 分步生成 → 中间检查 → 迭代优化 → 最终渲染

**实现架构**：

```
用户输入
    ↓
Agent Planner（规划阶段）
  - 分析需求复杂度
  - 决定是否需要分步
  - 拆解为子任务列表
    ↓
Step Executor（执行阶段）
  - 按子任务逐个调用 LLM
  - 每步生成部分元素
  - 中间结果合并到画布
    ↓
Quality Checker（检查阶段）
  - 检查元素数量是否合理
  - 检查箭头连接是否正确
  - 检查布局是否美观
  - 如有问题，触发修复步骤
    ↓
Final Optimizer（优化阶段）
  - 整体布局优化
  - 样式统一
  - 导出最终结果
```

**支持的 Agent 模式**：

| 模式 | 说明 | 适用场景 |
|------|------|----------|
| **骨架填充** | 先生成节点骨架 → 再填充细节 → 最后添加连接 | 大型架构图（20+ 节点） |
| **分区域生成** | 按功能区域分批生成 → 合并 → 统一布局 | 微服务架构、系统设计 |
| **迭代精化** | 生成初稿 → 识别问题 → 修复 → 重复直到满意 | 复杂流程图 |
| **参考图生成** | 分析参考图片 → 提取结构 → 生成类似图表 | 从截图重建图表 |

**关键技术点**：
1. **Agent Loop**：实现 `plan → execute → check → fix` 的循环
2. **工具注册**：给 Agent 提供工具（如 `addElements`、`moveElement`、`changeColor`）
3. **中间状态管理**：维护当前画布状态，每步增量更新
4. **成本控制**：限制最大步数和 token 消耗
5. **进度反馈**：SSE 流式返回每步的执行状态

**价值**：这是项目最大的差异化竞争力。当前市场上的 AI 图表工具都是一次性生成，Agent 模式能处理复杂得多的场景。

**工作量**：3-4 周

---

### 3.2 代码仓库图表生成

**描述**：分析代码仓库的结构，自动生成架构图、依赖关系图、ER 图等。

**实现思路**：
1. 用户输入 GitHub 仓库 URL 或本地路径
2. 后端克隆/读取代码，使用 AST 解析器分析：
   - 目录结构 → 架构图
   - import/require 依赖 → 依赖关系图
   - 数据库模型定义 → ER 图
   - API 路由定义 → API 流程图
   - 类定义 → 类图
3. 将分析结果作为上下文发送给 LLM
4. LLM 生成对应的图表代码

**支持的分析类型**：
| 输入 | 输出图表 |
|------|----------|
| 目录结构 | 分层架构图 |
| package.json / requirements.txt | 依赖关系图 |
| Prisma/SQLAlchemy 模型 | ER 图 |
| Express/FastAPI 路由 | API 流程图 |
| TypeScript 接口定义 | 类图 |
| Docker Compose | 部署架构图 |

**价值**：开发者经常需要为现有项目绘制架构图，手动绘制耗时且容易过时。

**工作量**：2-3 周

---

### 3.3 图表知识库

**描述**：将用户的图表、模板、代码片段组织为可检索的知识库。

**实现思路**：
- 新建 `knowledge_base` 表，存储结构化的图表元数据
- 对图表内容进行向量化嵌入（使用本地 embedding 模型或 API）
- 支持语义搜索："找一下上次画的那个微服务架构图"
- 支持标签分类和文件夹组织
- 支持导出为 Markdown 文档

**价值**：随着图表数量增长，用户需要高效地检索和复用历史图表。

**工作量**：2-3 周

---

### 3.4 图表自动更新

**描述**：当源数据变化时，自动重新生成图表。

**应用场景**：
- 监控架构图：从 Prometheus/Grafana 的指标自动更新服务健康状态
- 数据库 ER 图：当数据库 schema 变更时自动更新
- API 流程图：当路由文件变更时自动更新

**实现思路**：
- 支持配置"数据源"（GitHub webhook、数据库连接、API 端点）
- 数据源变化时触发重新生成
- 使用 diff 算法对比新旧版本，仅更新变化部分

**价值**：技术文档中的图表经常因为代码变更而过时。

**工作量**：3-4 周

3. 图表对比视图 依赖于 版本历史（需要多个版本的数据）
4. 自定义 AI 操作 依赖于 代码片段库（片段可作为操作的参考）
