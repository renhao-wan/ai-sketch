# AI 响应缓存

## 简介

AI Sketch 内置了智能响应缓存机制，能够记住之前生成过的图表代码。当您输入相同的描述时，系统会直接返回缓存的结果，无需等待 AI 重新生成。

## 工作原理

### 缓存条件

缓存功能在以下条件下自动生效：

1. **纯文本输入** — 上传图片的请求不会被缓存
2. **首次生成** — 重新生成（点击"重新生成"按钮）不会使用缓存
3. **单轮对话** — 只有新对话的第一条消息会触发缓存查找

### 缓存键

系统使用以下信息组合生成唯一的缓存标识：

- 用户输入的描述文本
- 图表格式（Excalidraw / Mermaid / Draw.io）
- 图表类型（流程图、架构图、思维导图等）

只有当这三个条件都完全匹配时，才会返回缓存结果。

### 缓存策略

| 参数 | 值 | 说明 |
|------|-----|------|
| 有效期 | 7 天 | 超过 7 天的缓存会自动清理 |
| 最大条目数 | 1000 | 超出时自动删除最久未使用的记录 |

## 用户体验

### 缓存命中时

当缓存命中时，您会看到：

1. 图表代码几乎瞬间出现（分块流式输出）
2. 控制台会显示 `[Generate] Cache hit` 日志

### 缓存未命中时

正常调用 AI 生成，生成结果会自动保存到缓存，供下次使用。

## 常见问题

### Q: 为什么我的请求没有使用缓存？

可能的原因：

- 上传了图片（图片请求不缓存）
- 点击了"重新生成"按钮
- 在已有对话中继续提问（多轮对话不缓存）
- 修改了图表类型或格式
- 距离上次生成超过 7 天

### Q: 缓存会不会占用太多空间？

不会。缓存存储在本地 SQLite 数据库中，每个条目只保存文本代码（不含图片），占用空间很小。系统会自动清理过期和多余的缓存。

### Q: 如何清除缓存？

目前没有提供手动清除缓存的界面。缓存会自动过期（7 天）或在超出上限时自动清理。

如果需要立即清除，可以：

- 删除数据库文件（`data/ai-sketch.db`）
- 通过 API 调用清除接口（开发者选项）

## 技术细节

缓存数据存储在本地数据库的 `response_cache` 表中：

| 字段 | 说明 |
|------|------|
| `prompt_hash` | 输入文本的哈希值 |
| `format` | 图表格式 |
| `chart_type` | 图表类型 |
| `response` | 生成的代码 |
| `created_at` | 创建时间 |
| `last_used_at` | 最后使用时间 |
| `use_count` | 使用次数 |
