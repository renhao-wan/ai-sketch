## 一、高严重问题（2 个）

### 1. `exportConfigs` 泄露 API Key 明文 🔴 需验证

- **文件**: `lib/db/config-manager.ts:307`
- **问题**: `getAllConfigs()` 自动解密 API Key，导出 JSON 包含明文密钥
- **修复方案**: 导出时对 API Key 脱敏（只保留前 4 位 + 后 4 位）
- **影响**: ⚠️ 导出文件内容会变化，用户导出后再导入会丢失完整 Key
- **回归测试**: 导出功能 → 导入功能 → 验证导入后 Key 是否可用（可能需要用户重新输入完整 Key）
- **状态**: ⬜ 待修复

### 2. 事务嵌套深度使用全局变量 🟡 低风险

- **文件**: `lib/db/transaction.ts`
- **问题**: `transactionDepth` 是模块级变量，异步事务交错时会干扰
- **修复方案**: 改为基于 `AsyncLocalStorage` 的上下文追踪，或在文档中约束 `fn` 必须同步
- **影响**: 当前所有调用方的 `fn` 都是同步的，改 AsyncLocalStorage 不会改变行为
- **回归测试**: 标签 CRUD、对话保存、配置保存等涉及事务的操作
- **状态**: ⬜ 待修复
